Claude Mobile Onboarding Flow

Permission-based feature discovery with graceful degradation

1
User Makes Request
User asks Claude to perform a complex task that requires calendar access: "Schedule a focus time block over the next work week that avoids conflicts with existing meetings. Show me some nearby coffee shops where I can work – and set a timer for a 90-minute deep work session!"
Key Insight: Claude accepts complex, multi-step requests naturally. The user doesn't need to know what permissions are required upfront.
2
Claude Identifies Need & Requests Permission
Claude analyzes the request, checks current time, and realizes it needs calendar access to find focus time slots. A permission dialog appears requesting calendar access.
Permission Request: Calendar - Search calendars and events

"Claude will search your calendar for events and appointments."

Options: Allow Once | Always Allow | Don't Allow

"Review each action carefully before approving."
Just-in-Time Permission: Permission requested exactly when needed, with clear context about why.
3
Graceful Degradation on Denial
If user denies calendar access, Claude doesn't fail – it adapts! Shows "Calendar access denied" status and explains what it can still do without the permission.
Graceful Handling: "I see I don't have access to your calendar yet. I can create a focus time block for you, but I'll need you to grant calendar access so I can check for conflicts and add it to your calendar."

Claude continues with what it CAN do:
  1. Set up that 90-minute timer for your deep work session
  2. Find nearby coffee shops where you can work
4
Secondary Permission Request
To find nearby coffee shops, Claude needs location access. Another permission dialog appears, this time for location.
Permission Request: Location - Approximate or precise

"Claude will use your location info to provide context-aware responses."

Options: Allow Once | Always Allow | Don't Allow
Progressive Permissions: Each permission requested separately, as needed for specific features. User maintains granular control.
5
Feature Execution with Available Permissions
With the permissions user granted, Claude executes what it can. In this case, it sets a 90-minute timer even without calendar or location access.
Always Useful: Even with denied permissions, Claude completes parts of the task it can do. The timer is set (1:30:00), showing immediate value regardless of permission status.

Design Principles

  • Just-in-Time Permissions: Request permissions exactly when needed, with clear context
  • Graceful Degradation: Never block the entire experience – do what you can with available permissions
  • Progressive Disclosure: Each permission requested separately, not all upfront
  • Transparent Communication: Clearly explain what each permission enables and what happens without it
  • User Control: Always offer "Allow Once" for users who want to try before committing
  • Immediate Value: Show value quickly, even with denied permissions
